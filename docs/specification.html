The Billt Language Specification

The "for" loop

The "for" loop looks like this: 

(for x~ = 1; < x 101; x += 1:
    print $x
)
The syntax is 4-part: 
(for assignments; expressionEvaluatingToBool; stmt; assignment; assert:
    ...body...
)
  1. Assignments are optional.
  2. The condition is the first non-assignment statement and must be an expression that evaluates to Bool. It is mandatory, but you can just write true to signify an empty condition
  3. The stepping/iteration change code is everything that comes after the condition and until the Intro, inclusive. (Intro is the statement terminated by a colon :)
  4. The body is everything after the colon and it may be comprised of arbitrary code

At least one of stepping code or body must be present in a well-formed loop.

Examples of syntactically correct loops:

(for x~ = 1; y~ = 100; < x 101; x += 1; y -= 1:
    print $x
)

(for ; true:
    print `Infinite loop`
)

Examples of incorrect loops:

(for x~ = 1; y~ = 100; < x 101:
)

(Neither stepping code nor body are present)

(for x~ = 1; x += 1:
    print $x
)

(The condition is absent)
